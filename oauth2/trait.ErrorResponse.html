Server Error Response

See Section 5.2 of RFC 6749. This trait exists separately from the StandardErrorResponse struct to support customization by clients, such as supporting interoperability with non-standards-complaint OAuth2 providers.

The Display trait implementation for types implementing ErrorResponse should be a human-readable string suitable for printing (e.g., within a RequestTokenError).
